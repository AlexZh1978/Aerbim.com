The "Fire Detectors" panel

Purpose: the "Fire Detectors" panel is intended for:

  • Selection of the category of equipment (fire detectors, control devices, power supplies, modules and components).

  • Search for the required BIM models (families) in accordance with the specified criteria.

  • Getting information about the found BIM models (families) (manufacturer, developer, LOD, type sizes, etc.)

  • Selection of the required BIM models (families) based on the results of the analysis.

  • Uploading and/or inserting selected BIM models (families) into the current Autodesk Revit document.

Terminology

For an unambiguous understanding of the principles of operation and the user interface of the add-in, it is necessary to understand in what sense the following terms are used.

BIM-model (building information model) is an object-oriented parametric 3D model that digitally represents the physical, functional and other characteristics of the product (or its individual parts) as a set of information-rich elements.

BIM-model search is the process of searching the database for the requested BIM model in accordance with certain search criteria.

BIM-model search criteria is a set of parameter values that the requested in the database BIM model must satisfy. Parameter values are set by selecting or entering in the interface elements of the dialog box.

BIM-model selection is the process of selecting the required BIM models by the user from the BIM models found in the database. The selection is made for subsequent loading and/or insertion by highlighting the names of BIM models and/or their type sizes in the interface elements of the dialog box.

Uploading BIM-models is the process of uploading BIM models to the current Autodesk Revit document. The download starts after clicking the corresponding button in the dialog box.

BIM-model Insertion is the process of inserting one of the selected BIM model (or type size) into the current Autodesk Revit document. The BIM model (or type size) is selected in the interface element of the dialog box.

Family is a group of similar elements that is characterized by a common set of properties and related graphical representations. Each family belongs to one specific category and receives a set of primary parameters from the category. The remaining parameters are set by the user (if necessary).

Embedded family is a downloadable family that is used inside another family with the ability to install dependencies, but without taking into account the specifications.

General family is a embedded family with the possibility of marking in the project and accounting in the specifications in the project.

Type size is a named set of parameter values.

LOD (level of development) is the level of elaboration which determines the completeness of the elaboration of the BIM model element. The level of elaboration sets the minimum amount of geometric, spatial, quantitative, as well as any attribute information necessary to solve modeling problems at a specific stage of the life cycle of a construction object.

Family category is a group of elements used to model a construction object: fire alarms, devices for calling and alerting, windows, doors, walls, ceilings, etc.

Collision is a logical, geometric or physical contradiction (or ambiguity) in the user's instructions, making it impossible to implement these instructions. Before implementing the instructions, the elimination of collisions is required.

Commands of the "Fire Detectors" panel

The "Fire Detectors" panel provides commands for performing the following actions:

  • Searching, selection, loading and/or insertion of BIM models (families) of smoke detectors into the project.
  • Searching, selection, loading and/or insertion of BIM models (families) of temperature detectors into the project.
  • Searching, selection, loading and/or insertion of BIM models (families) of manual call points into the project.
  • Searching, selection, loading and/or insertion of BIM models (families) of flame detectors into the project.
  • Searching, selection, loading and/or insertion of BIM models (families) of fire alarm control panels into the project.
  • Searching, selection, loading and/or insertion of BIM models (families) of fire alarm power supplies into the project.
  • Searching, selection, loading and/or insertion of BIM models (families) of fire alarm modules and components into the project.

Note

In the process of working with categories of equipment, when searching for BIM models (families) contained in the database, the following situations may occur:

  1. Some equipment may belong to different categories of the "Fire Detectors" panel. Accordingly, this must be taken into account when searching for the required BIM model (family).

Example: The «ARBM_Извещатель_КомбинированныйРадиоканальный_АРГУС-СПЕКТР_Аврора-ДТ-ПРО_L2_Ru_11.rfa» family models a combined radio-channel fire detector (heat + smoke). Accordingly, this family can be included in the sample result both when searching for BIM models (families) of smoke detectors and when searching for BIM models (families) of heat fire detectors.

  1. Some equipment may belong to different categories related to the "Fire Alarm" and "Fire Detectors" panels. Accordingly, this must be taken into account when searching for the required BIM model (family).

Example: The «ARBM_Контроллер_РадиоканальныйУстройств_АРГУС-СПЕКТР_Табло-РР-ПРО_L4_Ru_11.rfa» family models a fire alarm light radio channel, which contains a radio channel controller (hereinafter called as RR), designed to increase the range of the ISS radio channel. Accordingly, this family can be included in the sample result both when searching for BIM models (families) of fire alarms, and when searching for BIM models (families) of modules and components from the "Fire Alarm" panel, as well as when searching for BIM models (families) of modules and components from the "Fire Detectors" panel.
