The "Adaptation" panel

Purpose: The "Adaptation" panel is intended for:

  • Adapting existing families to work with the 3D Security Designer add-in with the ability to adapt nested families.

  • Adapting existing families to work with the «ARBM_ШаблонПроекта_СПС-СОУЭ_R2019_v1.1.rte» template.

  • Downloading the «ARBM_ШаблонПроекта_СПС-СОУЭ_R2019_v1.1.rte» template.

  • Downloading a work guide with the «ARBM_ШаблонПроекта_СПС-СОУЭ_R2019_v1.1.rte» template.

Terminology

For an unambiguous understanding of the principles of operation and user interface of the add-in, it is necessary to understand in what sense the following terms are used:

Adaptation of families is an additional automatic parameterization of families.

Adaptation to the add—in is an additional automatic parameterization of families to ensure work with the 3D Security Designer add-in.

Adaptation to the template is an additional automatic parameterization of families to ensure work with the «AER_ШаблонПроекта_СПС-СОУЭ_R2019_v01.rte» template.

Embedded family is a downloadable family that is used inside another family with the ability to install dependencies.

Family is a group of similar elements that is characterized by a common set of properties and related graphical representations. Each family belongs to one specific category and receives a set of primary parameters from the category. The remaining parameters are set by the user (if necessary).

Parametrization is the process of assigning parameters to objects (families).

Parameter is a property of Autodesk Revit families. The parameter can be system (set for a specific category at the software level) or user (created by the user). The parameters are used to store and display information about the family. The parameters allow you to change the family without having to edit it in the Family editor. The information can be entered into the parameter during the development of the family by the user, or calculated automatically based on data from the model.

Project template is a file with the RTE extension containing a number of settings for project parameters and families used to create new projects.

Add—in (plugin) is an independently compiled software module that is dynamically connected to the main program and designed to expand and/or use its capabilities.

Collision is a logical, geometric or physical contradiction (or ambiguity) in the user's instructions that makes it impossible to implement these instructions. Before implementing the instructions, the elimination of collisions is required.

Commands of the "Adaptation" panel

The Adaptation panel provides commands for performing the following actions:

  • Adaptations of families to the plugin.
  • Adaptation of families to the template.

Note

The commands from the "Adaptation" panel work only for the following categories of Revit families (in the Autodesk classification):

  • "Fire alarm".
  • "Security alarm system".
  • "Sensors".
  • "Electrical equipment".
  • "Power electrical appliances".
  • "Communication devices".
  • "Lighting devices".
  • "Devices for calling and notification".
  • "Telephone devices".
