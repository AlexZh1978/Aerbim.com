The "Sound Alarm" command

Purpose: searching, selection, loading and placement of sound alarm in the project.


Fig. 1. Launching the "Sound Alarm" command

The command works only on the following types: types of floors, types of ceilings, types of facades, types of sections, 3D views. To run the command, click the "Sound Alarm" button in the "Fire Alarm" group on the "Fire Alarm" panel of the "3D Security Designer" tab. After that, the window "Choose, Download and Place BIM models (families) of Sound Fire Alarm" will be displayed (Fig. 2).


Fig. 2. The window "Choose, Download and Place BIM models (families) of Sound Fire Alarm"

  1. If the "Specify Code" checkbox is selected, then this allows you to enter the article value for the desired BIM models (families) of the alarm sounder in the text field. At the same time, only one entry can be selected in the list of manufacturers and the "All" button is not active.


Fig. 3. Entering the value of the article

  1. The drop–down list "Language (localization)" allows to select the language for describing parameters in BIM models (families).


Fig. 4. Choosing the language of parameter description

  1. The "Revit Version" drop-down list displays the current version of Autodesk Revit.


Fig. 5. Information about the current version of Autodesk Revit

  1. The "Manufacturer" list contains a list of manufacturers of sound alarm.


Fig. 6. List of manufacturers

  1. The "All" button - allows you to select all manufacturers at once. The button is active only if the "Specify Code" checkbox is not selected and there is no selection of all the manufacturers in the list at once.


Fig. 7. Choosing one of the manufacturers


Fig. 8. Selection of all the manufacturers

  1. The "Nothing" button cancels the selection. The button is active only if at least one manufacturer is selected.
  2. The "Search" button - starts the process of searching for BIM models (families) in the database. The button is active only if at least one manufacturer is selected.


Fig. 9. Search process indicator

  1. The "Search Results" list contains information about BIM models (families) found in the database and allows you to select them for subsequent download and/or insertion into the current Autodesk Revit document. At the same time, in case of subsequent selection of another manufacturer(s), to update the list of "Search Result", click the "Search" button. If this is not done, then this list will contain information that is irrelevant for the newly selected manufacturer(s).


Fig. 10. The result of selecting families from the database

The parameters contained in the "Short Description" field are shown in Table 1:

Table 1
Name of the parameter Value

Developer Contains a link to the website of the developer of the BIM model (family)
  1. The "Show types" button - opens a list with types for the selected BIM model (family). This list contains detailed information on types and allows you to select them. If the selected BIM model (family) does not have types, the "Show types" button is not displayed.


Fig. 11. "Show types" button


Fig. 12. List of types for the selected family

  1. The "All" button - allows you to select all BIM models (families) and their types at once. At the same time, it should be noted that if the selected family has types, of which none has been selected, then this family as a whole, and accordingly its types, will not be included in the list of downloadable and/or inserted families. Therefore, in this case, for correct loading and/or insertion, at least one type size must be allocated for the selected family.
  2. The "Nothing" button cancels the selection. The button is active only if at least one BIM model (family) is selected.
  3. If the "Download" switch is set, then the selected BIM models (families) will be loaded into the current Autodesk Revit document.
  4. If the "Download and Place" switch is set, then the selected BIM models (families) will be downloaded and inserted into the current Autodesk Revit document. In this case, all the selected BIM models (families and/or types) will be loaded, and the family (type size) that is currently selected for insertion in the drop–down list will be inserted.


Fig. 13. Selecting the "Download and Place" switch

  1. The "OK" button - starts the process of downloading and/or inserting BIM models (families) of sound alarm into the current Autodesk Revit document. The button is active only if you select at least one family that does not have types and/or at least one type size for a family with types.
  2. The "Cancel" button cancels the "Sound alarm" command and closes the "Choose, Download and Place BIM models (families) of Sound Fire Alarm" window. After searching by the specified criteria and selecting the required BIM models (families) of sound alarm within the "Sound alarm" command in the "Choose, Download and Place BIM models (families) of Sound Fire Alarm" window and clicking the "OK" button, the process of downloading and/or inserting them into the current Autodesk Revit document starts. After the end of this process, the downloaded family will appear in the "Project Manager" window in the "Families" section (Fig. 14).


Fig. 14. Displaying the downloaded family in the "Project Manager" window

If, at the same time, the "Download and Place" switch was set in the "Choose, Download and Place BIM models (families) of Sound Fire Alarm" window, then after loading, the selected family will also be inserted in the current view, provided that this view satisfies the working conditions of the "Sound Alarm" command. This command offers the user the appropriate insertion options (for example, to set a plane, a face) using the Autodesk Revit interface. The process of inserting a BIM model (family) is shown in fig. 15.


Fig. 15. The process of inserting a family instance into the current view (ceiling plan in this example)

The operation of the command after inserting an instance of the "24 B" type size of the "BC_Оповещатель_Адресный_Болид_С2000-ОПЗ_Ru_11" family is shown in Fig. 16.

Note: if the window "Choose, Download and Place BIM models (families) of Sound Fire Alarm" is closed by clicking on the "OK" button, then the next time this window is opened, its previous settings will be saved in the current Autodesk Revit session. And if the above window is closed by pressing the "Cancel" or "Close" buttons (in the upper right corner of the window), then the settings will not be saved and they will have to be set again the next time it is opened.


Fig. 16. Inserted copy of the "24 В" type size of the family "BC_Оповещатель_Адресный_Болид_С2000-ОПЗ_Ru_11" on the ceiling plan

The following situations and ambiguities are possible when working with the "Sound alarm" command:

1. If after clicking on the "Sound Alarm" button in the "Fire Alarm" group on the "Fire Alarm" panel, when the "Choose, Download and Place BIM models (families) of Sound Fire Alarm" window opens, there is no connection to the server, then the corresponding information message will be displayed (Fig. 17), defining further actions of the user, namely, to retry the connection or not. In case of failure, the window "Choose, Download and Place BIM models (families) of Sound Fire Alarm" will close, the command will be completed and no changes will be made to the document.


Fig. 17. Informational message about the absence of a connection to the server

  1. If during the work of the team the connection to the database of BIM models (families) breaks, the corresponding information message will be displayed (Fig. 18).


Fig. 18. Informational message about loosing connection to the database of BIM models (families)

  1. If during the upload and/or insertion process it is determined that any of the uploaded/inserted BIM models (families) has already been uploaded to the current Autodesk Revit document, the corresponding information message will be displayed (Fig. 19), prompting the user to reload it into the document.


Fig. 19. Informational message about the possible reloading of the BIM model (family) into the document

  1. If there are no BIM models (families) in the database that meet the search criteria, then the corresponding information message will be displayed (Fig. 20).


Fig. 20. Informational message about the absence of BIM models (families) in the database that meet the search criteria

  1. If for some reason it is not possible to load and/or insert the selected types, the corresponding information message will be displayed (Fig. 21). At the same time, if, in the context of executing a command, the insertion of families is required and one of the types that was defined for insertion was not loaded, then the insertion of families into the document will not occur and the command will be completed.


Fig. 21. Informational message about the failure to load the selected types into the current document

  1. In case of an unexpected failure of the command, the corresponding information message will be displayed (Fig. 22), while no actions will occur.


Fig. 22. Informational message about the occurrence of a failure when executing a command
