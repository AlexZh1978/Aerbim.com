The "Marking" panel

Purpose: the "Marking" panel is designed for automatic marking of elements logically connected in a circuit. The panel allows you to work with such elements at the level of circuits, groups of elements and individual elements.

Marking elements, i.e., assigning a string value to each element that identifies this element in the project, is one of the most routine operations. Marking is complicated by the fact that during the design process, circuit elements can be either added or removed. This often happens with fire and burglar alarm detectors.

The BIM methodology involves automating such routine processes to avoid collisions and accelerate design, which is achieved with the help of this panel.

Terminology

For an unambiguous understanding of the principles of operation and user interface of the add-in, it is necessary to understand in what sense the following terms are used:

Mark (noun) is a string value with a user–defined structure that identifies an instance of the Revit family as part of the project, as well as identifying the product corresponding to the family in the design / installation / commissioning, etc. documentation.

Mark (verb) means to apply marking

Marking is the process of manually or automatically correlating specific Revit families with specific marks.

Mark format is a user–defined marking structure. As a rule, the mark contains alphanumeric characters, ordinal numbers, separator characters, etc. in some order.

Visualization of marking is the transmission by visual means of information about the marking of the family in the project environment, as well as about the marking of the product corresponding to the family in the design / installation / commissioning, etc. documentation.

Circuit is a subset of instances of Revit families in the project, selected by the user, and having the order specified by the user. The order of the families can be used both for marking and for further design of low-current electrical systems, including automated ones.

Collision is a logical, geometric or physical contradiction (or ambiguity) in the user's instructions, making it impossible to implement these instructions. Before implementing the instructions, the elimination of collisions is required.

"Marking" panel commands

The "Marking" panel provides commands to perform the following actions:

  • Marking the circuit according to the pre-selected mark format. Labeling takes into account the presence of embedded families and the multiplicity of addresses for a single device (in the case of BIM-design of address security systems).
  • Changes to the existing marking in the circuit.
  • Deleting circuit marks.
  • Getting information about the marking of the circuit.
  • Adding marked elements to the circuit.
  • Deleting marked elements from the circuit.
  • Alternative mark assignments for circuit elements.
  • Obtaining information about the mark of the circuit element.
  • Searching for circuit elements by mark value.
  • Creation of mark formats.
  • Mark formats management.
  • Mark visualization management.

Note

The commands from the Marking panel work only for the following categories of Revit families (in the Autodesk classification):

  • "Fire alarm".
  • "Security alarm system".
  • "Detectors".
  • "Electrical equipment".
  • "Power electronics".
  • "Communication devices".
  • "Lighting devices".
  • "Devices for calling and notification".
  • "Telephone devices".

The marked families must be adapted to the plugin. The adaptation process consists in creating special family parameters. The adaptation process can be automated using the "Adaptation" panel as part of the plugin.
