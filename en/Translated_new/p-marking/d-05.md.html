The "Add Elements" command

Purpose: adding elements (families) to an existing circuit is used to insert elements into the specified location of an already existing circuit with automatic updating of the order, marking and visual representation of elements mark in the circuit.


Fig. 1. Launching the "Add Elements" command

To run the command, click the "Add Elements" button in the "Actions with Groups of Elements" group on the "Marking" panel of the "3D Security Designer" tab. If no circuits have been created in the current document before, a window will be displayed informing the user about this.

Otherwise, the "Add Elements to Existing Circuit" window will be displayed.


Fig. 3. The "Add Elements to Existing Circuit" window

The window control elements are responsible for the following functionality:

  1. The "Circuit Name" list allows you to select one of the circuits previously created in the document to add elements (families) to it.
  2. The "Circuit Mark Format" field gives an idea of the mark format for the selected circuit. The field contains the format name and format structure separated by a colon.
  3. If the "Add Before Specified Circuit Element" checkbox is selected, then the addition of elements will occur before the specified element in the circuit. If not, then after.
  4. The "Element Marking Statistics" table contains quantitative information about the selected circuit, including the fields:
  • "Number of elements" is the amount of elements in the circuit, including embedded ones.
  • "Counter Values in Mark Format" is the maximum amount of numbers provided by mark format.
  • "Unused Counter Values" is the amount of unused numbers that can be assigned to elements without violating the requirements of mark format.
  • "Out-of-Range Counter Values" is the amount of elements that lie outside the limits provided by mark format.
  1. The "Add Elements" button starts the selection of items to be added to the selected circuit.
  2. The "Cancel" button cancels the "Add Elements" command and closes the "Add Elements to Existing Circuit" window.

After setting up the logic of the "Add Elements" command in the "Add Elements to Existing Circuit" window and clicking the "Add Elements" button, the process of adding elements to the selected circuit starts.

All elements included in the circuit will be highlighted in green. The status bar will display the query "Specify an element in the circuit relative to which new elements will be added". After selecting this element, it will be highlighted in red and the status bar will prompt "Select elements to be added to the circuit".

Next, select the elements to be added to the selected circuit and click the "Done" button on the parameters panel to complete the selection. As a result, new elements are added to the circuit, with automatic updating of the order, marking and visual representation of the marking of elements in the circuit. All elements of the rebuilt circuit will be highlighted.

Keep in mind that you can only add families of certain categories. The added families must be adapted to work with the "3D Security Designer" add-in.

The following ambiguities are possible when working with the "Add Elements" command:

If, during the process of adding elements to the circuit, the selected elements contain elements that were previously included in other circuits, then the corresponding information window will be displayed, determining the user's further actions. At the same time, the window will display information about the number of elements and the names of the circuits for which there was an ambiguity. If the command continues to work, the numbering of the elements and the visual representation of the marking in the affected circuits will automatically change (if visualization for them was provided earlier) in accordance with the last settings made in the Customize Mark Visualization window.


Fig. 11. Elimination of the collision

If, during the process of adding elements to the circuit, the number of selected elements exceeds the number provided by the mark format, the corresponding information window will be displayed, determining the user's further actions. At the same time, information about the number of actually selected elements and the maximum number of elements determined by the mark format will appear in the window.


Fig. 12. Elimination of the collision

If the command fails, a corresponding message will be displayed, while no elements will be added to the circuit.
