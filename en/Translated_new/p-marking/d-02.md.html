The "Change Circuit" command

Purpose: changing elements marks in existing circuit is applied if:

  • you need to use another mark format;
  • you need to change the values for parameters that influence the marks.


Fig. 1. Launching the "Change Circuit" command

To run the command, click the "Change Circuit" button in the "Actions with Circuits" group on the "Marking" panel of the "3D Security Designer" tab. If no circuits have been created in the current document before, a window will be displayed informing the user about this.


Fig. 2. Warning about the absence of circuits

Otherwise, the window "Change Marks in Existing Circuit" will be displayed.


Fig. 3. Window "Change Marks in Existing Circuit"

The window control elements are responsible for the following functionality:

  1. The "Circuit Name" drop-down list allows you to select one of the circuits previously created in the document to change it.

  2. The string representation "Circuit Mark Format" gives an idea of the mark appearance. The string contains the name and format structure separated by a colon.

  3. The button "Information on Existing Circuits" allows you to open a window reflecting information about marked circuits in the document.

  4. The "Format Selection" drop-down list allows you to select one of the formats previously created in the document for the circuit that is changed.

  5. The "Create Format" button allows you to open a window to create a new format in the document for the circuit that is changed.

  6. A list containing "Parameter" - "Value" pairs set by the user. These parameters include:

  • "Circuit name", the value of this parameter is entered by the user and must be unique within the document.
  • Other user parameters provided by the selected mark format. The values of these fields are not subject to uniqueness requirements.

The values for the above parameters can be changed by the user. If the mark format has been changed, then for parameters with the same names, the values are transferred so that they do not have to be entered again.

  1. If the "Mark Embedded Elements" checkbox is selected, then the embedded families are marked. At the same time, embedded families should be "General" with respect to the parent family, have a specific category and be adapted to work with the 3D Security Designer add-in.

  2. If the "Visualize Marks" checkbox is selected, the mark is visualized in accordance with the visualization settings.

  3. If the "Delete Alternative Marks" checkbox is selected, then when marking an element (family) the value of the parameter (if it was previously set) of the alternative mark will be deleted.

  4. If the check box "Count Addresses in Elements" is selected, then in the process of marking when forming the counter, the value of the parameter containing the number of addresses will be taken into account (if the device is addressable).

  5. The button "Customize Mark Visualization" allows you to open a window for configuring the visualization.

  6. The "Change Marks" button starts the process of changing marks for the existing circuit. The button is active only if the values for all parameters listed in paragraph 3 are filled in.

  7. The "Cancel" button cancels the "Change Circuit" command and closes the "Change Marks in Existing Circuit" window.

After setting up the logic of the "Change Circuit" command in the "Change Marks in Existing Circuit" window and clicking the "Change Marks" button, the process of changing marks of the previously selected circuit starts in accordance with the specified settings. If an alternative mark has been set for any elements in the circuit, then it will take precedence during visualization.

The following ambiguities are possible when working with the "Change Circuit" command:

If in the process of changing the circuit, the number of elements in the circuit has exceeded the number provided by the changed mark format, then the corresponding information window will be displayed, determining the user's further actions. At the same time, information about the number of actual elements in the changeable circuit and the maximum number of elements determined by the selected mark format will appear in the window, and the visual representation of the marking of the changeable circuit will automatically change (if visualization for it was provided earlier) in accordance with the last settings made in the "Customize Mark Visualization" window.

In case of a command failure, a corresponding message will be displayed, while there will be no change in the marking of the circuit.
